[2. RESUMEN DEL PROYECTO "CAMINSIDE"]
--------------------------------------------------------------------------------
"CamInside" es un ecosistema de software distribuido diseñado para desmitificar
el funcionamiento de la cámara en dispositivos móviles a nivel de software. 
El sistema no es una aplicación aislada; es un puente de datos en tiempo real 
que extrae metadatos, estados del ciclo de vida y buffers de imágenes de un 
smartphone real y los proyecta instantáneamente en un dashboard web interactivo 
(que sirve como diapositiva de la exposición) a través de WebSockets.

[3. ARQUITECTURA GENERAL Y TECH STACK]
--------------------------------------------------------------------------------
El sistema está completamente diseñado a nivel lógico y se compone de:

1. CLIENTE MÓVIL (Célula Móvil - Integrantes 3, 4, 5):
   * Stack: Android SDK Nativo, Kotlin, Jetpack Compose, Jetpack CameraX API.
   * Función: Interrogar al CameraManager, capturar eventos de la Activity, 
     separar los hilos de renderizado (Preview Surface), disparo (ImageCapture) 
     y procesamiento continuo (ImageAnalysis en formato YUV_420_888).

2. MIDDLEWARE SERVER (Célula de Integración - Integrante 2):
   * Stack: Node.js, Express, Socket.io.
   * Función: Levantar un servidor HTTP/WS en la red local (LAN) para actuar 
     como broker de eventos. Recibe JSONs del teléfono y los retransmite a la web.

3. DIAPOSITIVA INTERACTIVA WEB (Célula Web - Integrantes 1, 6, 7):
   * Stack: HTML5, Tailwind CSS, JavaScript (ES6), Reveal.js, Three.js (WebGL).
   * Función: Sustituir las diapositivas tradicionales por una SPA que muta su 
     DOM en tiempo real. Muestra consolas de logs asíncronas y renderiza un 
     modelo 3D despiezado del hardware que reacciona a los comandos del móvil.
   * Inteligencia Artificial: Integra Google ML Kit en la periferia para 
     visión artificial avanzada (Lectura de QR / Rostros a 30 FPS).
